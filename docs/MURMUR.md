# MURMUR — stochastic reverb (Hikari clone)

A reverb whose **room changes every time you play a note**. The tail is generated by an **8×8
Householder feedback delay network** (`suite_core::fdn::Fdn8` — the reusable suite FDN core,
shared with UNDERTOW / SEANCE / CHAMBER), and an **onset detector re-randomises the whole room**
— delay lengths, input diffusion, and damping color — on every transient. Two FDN instances
**ping-pong** with a 50 ms equal-power crossfade so the room swap is click-free. Zero reported
latency (a reverb is a time-smearing effect, not a fixed processing delay).

```
 in ─┬─ onset detector (band-energy rise, Sensitivity) ─── trigger ───┐  ┌── manual RE-ROLL
     │                                                                 ▼  ▼
     ├─▶ Fdn8 #A ─┐   on trigger: draw a NEW random room into the idle FDN, then
     └─▶ Fdn8 #B ─┤   50 ms equal-power crossfade to it (both FDNs always run, so the
                  │   idle one is pre-warmed and already holds the incoming transient)
                  └──▶ width ─ DC block ─ safety clip ─ wet
 in ──────────────────────────────────────────────────────► out = dry + Mix·(wet − dry)
```

## The FDN core (`suite_core::fdn::Fdn8`)

Eight delay lines feed back through an **8×8 Householder matrix** `H = I − (2/N)·vvᵀ/(vᵀv)` with
`v = ones`. For `v = ones` this is the **O(N) "sum trick"**: `(Hx)_i = x_i − (2/N)·Σx` — no
matrix multiply, just one sum. `H` is a real **orthogonal reflection** (`‖Hx‖ = ‖x‖`), so the
loop is **lossless** apart from the two things that shape the tail:

- **Per-line decay gain** `g_i = 10^(−3·Lᵢ/(RT60·sr))`, where `Lᵢ` is line `i`'s length. Every
  line — regardless of its length — reaches exactly **−60 dB after `RT60` seconds**, so the
  broadband decay equals the **Decay** setting.
- **Per-line one-pole damping** (a lowpass in each feedback path). DC passes at unity, so low
  frequencies decay purely by `g` while highs are rolled off — `Color` tilts this bright→dark
  with a small per-line spread so the tail has character.

The **input** is diffused through a **4-allpass chain per channel** (coefficient settable) and
injected with alternating signs (L → even lines, R → odd lines). The **output** taps sum even
lines to L and odd lines to R (fixed sign pattern) for a stereo field. Each line is a ring
buffer **preallocated at the maximum drawable length**, so `set_delays` changes the active
length at runtime with **no allocation** (safe under `assert_process_allocs`).

## Re-randomisation (the "stochastic" part)

Both FDNs process the input **all the time**, so the idle one is always pre-warmed with its own
random room and already contains the transient that is about to trigger the swap. On an onset
(or the **Re-Roll** button) MURMUR:

1. crossfades the output from the current FDN to the pre-warmed idle one over **50 ms**
   (equal-power, `cos/sin`), then
2. once the crossfade completes, **resets the now-idle FDN and draws a fresh random room** into
   it for next time.

Each **draw** (`suite_core::testsig::Rng`, deterministic) sets:

- **8 delay lengths** — the nominal lengths are spread geometrically across a 15–70 ms window
  scaled by **Size**, then each is perturbed by up to ±40 % × **Randomness**, then **nudged
  mutually prime-ish** (each forced odd, then bumped until it shares no factor with the others)
  to avoid the metallic flutter of commensurate delays;
- **damping color** = **Color** ± 0.5 × **Randomness**;
- **input-diffusion coefficient** = 0.55 ± 0.3 × **Randomness**.

At **Randomness = 0** every draw is the *identical deterministic nominal room*, so the RT60 is
well defined by **Decay** (done-bar 2). As **Randomness** rises, two identical hits land in
genuinely different rooms and their tails decorrelate (done-bar 1).

## Onset detector

A zero-latency, spectral-flux-style detector: the input is high-passed (800 Hz) to emphasise
transients, and a **fast envelope** (0.5 ms attack) is compared to a **slow envelope** (250 ms
release). An onset fires when `fast > ratio · slow` (plus an absolute floor), guarded by a
**120 ms refractory** gap so one transient can't double-trigger. **Sensitivity** lowers the
ratio from 3.0 (only big transients) to 1.2 (very twitchy).

## Freeze

**Freeze** sends both FDNs' RT60 to (near-)infinity — the per-line gain is capped just below 1.0
so the orthogonal loop sustains without ever growing — and **ducks the input to zero** (30 ms
smoothed) so the current wash is held as an infinite pad instead of being overwritten. Onsets
are ignored while frozen. (Factory presets never engage freeze — an eternal space is emulated
with a long **Decay** instead — because freeze from an empty buffer renders silent.)

## Live parameter behaviour

**Decay** and **Freeze** apply to both FDNs **live** every block (a gain change, click-free).
**Color** applies live only when the knob actually moves, so the per-draw color randomness
survives a still knob. **Size** and **Randomness** shape the *next* draw (they change the room,
so they take effect on the next onset / re-roll rather than glitching the current tail). **Width**
(mid/side on the wet), **Mix** (per-sample smoothed), and a wet-path **DC blocker** + knee'd
**safety clip** (exact identity below 0.9, so `mix = 0` — which uses the dry path — is untouched)
finish the signal.

## Parameters

| Group | Param | Range | Notes |
|---|---|---|---|
| Room | Size | 0–100 % | Scales the delay-length range (room size). Applies on the next draw. |
| Room | Decay | 0.2–20 s | RT60. Applied live to both FDNs. |
| Room | Color | −1…+1 | Damping tilt: −1 bright, 0 mild, +1 dark. |
| Room | Randomness | 0–100 % | How far each draw deviates from the nominal room. 0 = fixed nominal (RT60 = Decay). |
| Trigger | Sensitivity | 0–100 % | Onset-detector sensitivity (higher = triggers on smaller rises). |
| Trigger | Re-Roll | button | Manually draws a new room + crossfades (any press = one re-roll). |

| Output | Mix | 0–100 % | Dry/wet. |

## Done bar (mechanical, PRD §4 + build brief)

- Universal: no NaN/inf, peak ≤ 0 dBFS, non-silent; `mix = 0` nulls against the dry < −80 dB.
- **(1)** Two identical impulses 2 s apart with **Randomness up** → the two tail waveforms'
  cross-correlation is **< 0.9** (different rooms). Also asserts the core actually re-rolled
  (≥ 4 room draws).
- **(2)** With **Randomness = 0**, the **measured RT60 is within ±25 % of the Decay setting**
  (Schroeder backward-integration fit of the impulse response), at two settings.
- **(3)** **Freeze sustains**: after charging the reverb and freezing, the output RMS over 5 s of
  silence stays **> −50 dBFS** (and finite, ≤ 0 dBFS).
- **(4)** **Crossfade click bound**: the max sample-to-sample delta around a re-roll is within
  **4×** the steady-state tail (a hard tap-switch would be 10–100×).

The FDN core carries its own suite-core tests: impulse → monotone windowed decay; measured RT60
within ±25 % at two settings; energy bounded under noise excitation; reset clears the tail;
longer RT60 decays slower.

## Try it in FL

Find more plugins → add **Qeynos MURMUR** on drums, a vocal, keys, or a bus. Load **Never The
Same Hall** and play a phrase — each note's tail is a slightly different room. **Percussion
Rooms** is tuned for drums (snappy, very responsive onset, high per-hit variety); **Grief Space**
is a dark mournful wash for pads; **Frozen Nave** is a huge bright cathedral. Raise **Randomness**
for wilder room-to-room variation, drop it to 0 for a fixed reverb whose RT60 = **Decay**. Hit
**Re-Roll** any time to draw a new room by hand, or tick **Freeze** to hold the current wash as
an infinite pad. **Size** sets the room scale, **Color** tilts bright→dark, **Width** spreads the
tail, **Mix** blends parallel. Zero latency.

## Freeze Mix

**Freeze Mix** (0–100%, default 100%) works alongside the **Freeze** toggle. Freeze stays a
toggle; Freeze Mix sets how much of the held/frozen texture you hear versus the live signal
while Freeze is engaged. At 100% it is the classic hard freeze (unchanged); lower it to blend
the live source back in so the freeze is a smooth crossfade rather than a sudden jump. The
blend is smoothed (~15 ms) and only active while Freeze is on.

## What It Is

A stochastic reverb whose room is redrawn on every onset — play the same note twice and each
tail lands in a subtly different hall. An 8×8 Householder FDN re-randomises its delay lengths,
diffusion, and damping on every transient, crossfading click-free between two rooms. Use it
where a static reverb sits dead: it makes pads breathe, drum spaces vary hit-to-hit, and vocals
haunt a slightly different space each line.

## Signal Flow

```
in ─┬─ onset detector (Sensitivity) ─ trigger ─┐        ┌ RE-ROLL (manual)
    │                                           ▼        ▼
    ├─► FDN room A ─┐  draw a NEW random room (Size · Randomness · Color) into the
    └─► FDN room B ─┤  idle FDN, then a 50 ms equal-power crossfade to it
                    └─► Width ─ Freeze / Freeze Mix ─ wet
in ─────────────────────────────────────────────► out = dry + Mix·(wet − dry)
```

## Controls

- **Size** — room scale; scales the delay-length window, 0–100 %. Applies on the next draw.
- **Decay** — RT60 tail length, 0.2–20 s, applied live to both rooms.
- **Color** — damping tilt from bright (−1) through mild (0) to dark (+1).
- **Randomness** — how far each new draw strays from the nominal room, 0–100 % (at 0 it is a fixed reverb whose RT60 exactly equals Decay).
- **Sensitivity** — onset-detector threshold, 0–100 % (higher re-rolls on smaller transients).
- **Freeze** — sends RT60 to infinity and ducks the input: hold the current wash as an eternal pad.
- **Freeze Mix** — while Freeze is engaged, the blend of the held texture against the live signal, 0–100 %.
- **Re-Roll** — button; manually draw a fresh room and crossfade to it any time.
- **Width** — stereo width of the wet, 0–100 % (0 = mono).
- **Mix** — dry/wet blend, 0–100 % (0 leaves the dry untouched).
