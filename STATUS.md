## NOTES
- WIRE (2026-07-07): reusable pattern for any codec/heavy-DSP-in-the-audio-thread plugin.
  (a) A real third-party codec (opus-rs) can run in `process()` if a full block's work
  fits the RT budget (bench it — 20ms Opus frame = 0.3% RT) AND you wrap the call in
  `nih_plug::util::permit_alloc(|| …)` so the workspace's assert_process_allocs tolerates
  the codec's internal Vec churn. Offline harness tests call the core directly (no alloc
  guard active), so they need no wrapper — only the plugin `process()` does.
  (b) Frame-rate ↔ block-rate adapter: fixed-capacity ring FIFOs (a `SampleFifo` that never
  reallocs) accumulate host samples into 20ms frames and buffer decoded output; seed the
  out-FIFO with one frame so the streaming reader never underflows. The codec's own
  algorithmic delay is absorbed inside the frame buffer ⇒ measure total latency with an
  impulse and REPORT the measured value (don't try to sum sub-latencies analytically).
