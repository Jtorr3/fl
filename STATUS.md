- SMUDGE (2026-07-07): reusable patterns for STFT per-frame spectral effects (SEANCE, CARVE,
  any bin-domain op). (a) **Exact per-op bypass** = skip the whole op when its smoothed amount
  ≤ 1e-5 (leave the complex spectrum untouched); with all ops skipped the wet path is the
  `suite_core::stft` identity reconstruction, which nulls against the FFT_SIZE-delayed dry
  **< −60 dB** (much tighter than mix=0). Prime the amount smoothers to their target on the
  first `configure` (a `primed` flag) so a base amount of exactly 0 stays exactly 0. (b) **Bin
  permutation drops level by exactly −6 dB** (= √(fft/hop) = 2 for 2048/512): shuffling bins
  DECORRELATES overlapping WOLA frames, so their overlap-add sums in power (×√overlap) not
  coherently (×overlap). This is INDEPENDENT of redraw rate (a fixed perm loses the same 6 dB).
  Fix = pre-multiply the permuted content by √(fft/hop). Keep DC + Nyquist fixed in the perm
  (STFT zeroes their imag on inverse; moving a big bin into DC loses energy). (c) **A bin-domain
  op cannot be energy/peak-bounded in the spectral domain alone** — an additive/feedback echo
  matched to input ENERGY still doubles the TIME-DOMAIN peak (crest factor). Two-layer guard:
  energy-normalise the echo DOWN to a decaying envelope of input frame-energy (attenuate-only,
  no ducking while input present, ~1 s tail release) for musical level control, PLUS a wet-path
  **safety soft-clip that is EXACT identity below 0.9** (tanh only above). The identity region is
  the trick: normal reconstruction sits well under 0.9, so the null/identity done-bars are
  untouched while loud resonant peaks are bounded < 1.0. A global tanh would break the −60 dB
  null (tanh(0.5) errs ~−28 dB); the knee'd clip does not. (d) **Chaos S&H must modulate amounts
  MULTIPLICATIVELY** (random gain ≤ 1), never additively — that keeps a base-0 amount at 0 so
  exact bypass survives full-depth chaos (verified as a done-bar). (e) Stereo: run L as the
  "primary" channel that advances all shared per-frame state (chaos draw, perm redraw, smoothed
  eff params, per-band tables) and R as "secondary" reusing it — L and R STFTs are hop-aligned
  (same sample count) so both fire on the same frame; borrow-split the core's fields (stft_l vs
  shared/chan_l/cfg are disjoint) so each channel's `stft.process(|spec| frame(...))` closure
  compiles under NLL. (f) Blur phase = EMBER's phase-vocoder advance, but blended along the
  SHORTEST ARC by blur dominance: `phase_out = wrap(ph + va·wrap(advanced − ph))`, va ramps 0→1
  as amount 0.5→1 — phase-continuous, no switch click. Nothing added to suite-core (all SMUDGE
  DSP local to the crate, reusing stft/OnePole/ScopedFtz/Rng) ⇒ no suite-wide revalidate.
- SWARM (2026-07-07): reusable patterns for granular / capture-buffer plugins (CLEAVE, SEANCE,

  monotonic `serial`**; spawn scans for a free slot then steals min-serial — alloc-free under
